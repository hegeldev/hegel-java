# Hegel for Java

> **Beta:** Hegel is in beta. Please report issues at
> <https://github.com/hegeldev/hegel-java/issues> and check
> <https://hegel.dev/compatibility> for engine compatibility.

> This implementation was generated by Claude and has not been extensively used in
> production. Please report anything that looks wrong.

[API docs](https://javadoc.io/doc/dev.hegel/hegel) · [hegel.dev](https://hegel.dev)

`hegel` is a property-based testing library for Java. It is based on
[Hypothesis](https://github.com/hypothesisworks/hypothesis), powered by the Hegel engine.

Instead of writing tests with hand-picked example inputs, you describe a *property* that should
hold for all inputs and let Hegel generate inputs to try to falsify it. When it finds a failing
input it automatically **shrinks** it to a minimal counterexample.

## Installation

Add the dependency with Maven:

```xml
<dependency>
  <groupId>dev.hegel</groupId>
  <artifactId>hegel</artifactId>
  <version>0.1.0</version>
  <scope>test</scope>
</dependency>
```

Hegel requires **Java 22+** (it uses the Foreign Function & Memory API). Pass
`--enable-native-access=ALL-UNNAMED` to your test JVM, e.g. with Surefire:

```xml
<plugin>
  <groupId>org.apache.maven.plugins</groupId>
  <artifactId>maven-surefire-plugin</artifactId>
  <configuration>
    <argLine>--enable-native-access=ALL-UNNAMED</argLine>
  </configuration>
</plugin>
```

**Nothing else to install.** The native engine, `libhegel`, is bundled inside the jar for every
supported platform (Linux and macOS, x86-64 and ARM64); the right one is unpacked automatically on
first use. Nothing is downloaded at runtime. To use your own build instead, set
`$HEGEL_LIBHEGEL_PATH` to an explicit library, or place it on the OS's standard library search path
(`LD_LIBRARY_PATH` on Linux, `DYLD_LIBRARY_PATH` on macOS); the bundled library is the fallback.

## Quickstart

Here is a sort function with a subtle bug — it accidentally drops duplicate elements:

```java
import java.util.ArrayList;
import java.util.List;
import java.util.TreeSet;

static List<Integer> mySort(List<Integer> xs) {
  return new ArrayList<>(new TreeSet<>(xs)); // oops: a TreeSet removes duplicates
}
```

A traditional unit test with a hand-picked input like `[3, 1, 2]` would pass. Let's write a
property instead: sorting must not change the number of elements.

```java
import static dev.hegel.Generators.integers;
import static dev.hegel.Generators.lists;
import static org.junit.jupiter.api.Assertions.assertEquals;

import dev.hegel.HegelTest;
import dev.hegel.TestCase;
import java.util.List;

class SortTest {
  @HegelTest
  void sortingPreservesLength(TestCase tc) {
    // The optional label names the value in the falsifying-example output.
    List<Integer> xs = tc.draw(lists(integers()), "xs");
    assertEquals(xs.size(), mySort(xs).size());
  }
}
```

Run it with `mvn test`. Hegel finds the bug and shrinks it to the minimal failing input. It prints
the shrunk draws (the optional label names each one; here we passed `"xs"`) and then fails the test
with the property's own assertion error:

```
xs = [0, 0];

org.opentest4j.AssertionFailedError: expected: <2> but was: <1>
```

`[0, 0]` is the simplest input that triggers the bug: two equal elements, which `mySort`
collapses into one. Hegel generated a random failing list, then shrank both the list length and
the element values until it could not make the counterexample any smaller.

## Writing tests

Use the `@HegelTest` annotation on a JUnit 5 method that takes a `TestCase`, or call
`Hegel.check` inside any test:

```java
import static dev.hegel.Generators.integers;
import static org.junit.jupiter.api.Assertions.assertEquals;

import dev.hegel.Hegel;
import org.junit.jupiter.api.Test;

class CommutativityTest {
  @Test
  void additionCommutes() {
    Hegel.check(tc -> {
      int x = tc.draw(integers());
      int y = tc.draw(integers());
      assertEquals(x + y, y + x);
    });
  }
}
```

See the [API documentation](https://javadoc.io/doc/dev.hegel/hegel) for a full tour of generators,
combinators, control functions, and settings — the `dev.hegel` package overview is the getting-started
guide.
